# PDF 技术选型决策书

> 基于 108 个国标 PDF（47% 文字层 + 53% 扫描件）的实测结论。

---

## 一、决策树

```
PDF 来了
│
├── 有文字层？（逐页提取字符数 > 0）
│   ├── 是 ──→ PyMuPDF 逐页提取全文
│   │           │
│   │           ├── 编码正常？→ pymupdf4llm → Markdown (Pipe Table)
│   │           │                    └── 配光曲线表可保留结构，但单元格有碎片
│   │           │
│   │           │                    ↓ LLM 参数提取 ✅
│   │           │
│   │           └── 编码异常？→ 标注"不可读"  ← GB 4785 类
│   │                              │
│   │                              └── 走 OCR 捞正文信息（标准号/归口单位等）
│   │
│   ├── 否 / 太稀疏
│   │
│   └──→ OCRmyPDF + Tesseract chi_sim
│          │
│          ├── 正文文本可读（~85% 准确）
│          ├── 表格数值不可靠（~60-70%，小数点缺失）
│          │
│          └── 定位器标记参数页 → **人工录入精确值**
│
└── 最终产出:
    ├── index.json（参数页定位，122→22页，节约 82%）
    └── 人工对照参数页提取精确数值
```

## 二、各工具实测对比

### 文本提取

| 工具 | 速度（59页） | 中文正确 | 表格 | Markdown | 推荐场景 |
|------|------------|---------|------|----------|---------|
| **PyMuPDF** | **0.12s** | ✅ | ❌ 纯文本 | ❌ | 全量文字提取 / 定位器输入 |
| **pymupdf4llm** | ~0.3s | ✅ | ⚠️ 有结构但碎片化 | ✅ Pipe Table | PDF→Markdown 首选 |
| pdfplumber | 6.16s | ✅ | ✅ 原生 | ❌ 需手拼 | 表格专项提取 |
| pdfminer | 13.28s | ✅ | ❌ | ❌ | 慢，不推荐 |
| pypdfium2 | **0.14s** | ✅ | ❌ | ❌ | PyMuPDF 平替 |

### OCR

| 工具 | 安装 | 中文正文 | 表格数值 | 推荐场景 |
|------|------|---------|---------|---------|
| **Tesseract + chi_sim** | `apt install`（需 sudo） | ~85% 可读 | ~60-70%（小数点丢失） | 正文信息提取 / 定位器输入 |
| EasyOCR（未测） | `pip install`（含 PyTorch ~500MB） | 预期更高 | 未测 | Tesseract 不够好的替代 |

### 参数页定位

| 工具 | 查全率 | 查准率 | 方式 |
|------|--------|--------|------|
| **locator（关键词规则）** | **100%**（GB 4599） | 58% | YAML 配置规则，可迭代 |
| 未来：ML 分类模型 | — | — | 通过 model_hook.PageDetector 接入 |

## 三、各场景推荐

| 你要做什么 | 推荐方案 | 理由 |
|-----------|---------|------|
| **PDF → 纯文本**（定位器输入） | PyMuPDF | 0.12s/59页，最快 |
| **PDF → Markdown**（含表格） | pymupdf4llm | 原生 Pipe Table，速度可接受 |
| **PDF → 精确参数值** | 定位器 + 人工录入 | OCR 精度不足，自动化不可靠 |
| **编码异常 PDF** | OCR → 捞正文信息 | 所有数字提取器全灭 |
| **批量 108 个扫描件** | OCRmyPDF → Tesseract | 正文信息可恢复，表格需人工 |
| **未来参数页识别升级** | model_hook 接入 ML 模型 | 接口已预留，不拆架构 |

## 四、已知限制

| 限制 | 影响 | 缓解方案 |
|------|------|---------|
| Tesseract 需 sudo 安装 | 环境不可脚本化 | 选 EasyOCR（pip 全自动） |
| 编码异常 PDF（GB 4785 类） | 所有数字提取器乱码 | 走 OCR 捞正文 |
| pymupdf4llm 表格碎片化 | LLM 提取可能丢行列 | 人工核对 / pdfplumber 补表 |
| 扫描件 OCR 表格精度 60-70% | 数值不可自动提取 | 定位器标记后人工录入 |
